# JIDRA Analysis Validator

Interactive chat-style script that compares Claude API token usage **with** and **without** JIDRA context.

## Features

- **Two API calls per query**: Makes parallel requests to Claude (one with JIDRA context, one without)
- **Real token metrics**: Captures actual input/output/thinking tokens from Claude API
- **Cost comparison**: Calculates USD cost for both approaches
- **Latency tracking**: Measures response time for each call
- **Table-based results**: Displays metrics side-by-side for easy comparison
- **Session history**: Tracks all queries and shows cumulative savings
- **Interactive loop**: Ask multiple questions, switch models, review results

```bash
pip install anthropic tabulate
```

## Usage

### Basic Usage (with Demo)
```bash
python3 validate_jidra_analysis.py
```

### With Graph File
```bash
python3 validate_jidra_analysis.py --graph /path/to/graph.jsonl
```

### With Graph File and Repository
```bash
python3 validate_jidra_analysis.py --graph /path/to/graph.jsonl --repo /path/to/repo
```

### With Custom Model
```bash
python3 validate_jidra_analysis.py --graph graph.jsonl --model claude-sonnet-4-6
```

### Command-Line Options
```
--graph PATH         Path to graph.jsonl file
--repo PATH          Path to the repository/codebase
--model MODEL        LLM model: claude-opus-4-7, claude-sonnet-4-6, or claude-haiku-4-5
                     (default: claude-opus-4-7)
```

## Metrics Captured

| Metric | Description |
|--------|-------------|
| **Input Tokens** | Tokens consumed by the model to read your query + context |
| **Output Tokens** | Tokens generated by the model in the response |
| **Thinking Tokens** | Extended thinking tokens (if supported by model) |
| **Cost** | Calculated USD cost based on model pricing |
| **Latency** | Time taken for Claude API to respond (seconds) |
| **Stop Reason** | Why the model stopped generating (`end_turn`, `max_tokens`, etc.) |

## Supported Models

- `claude-opus-4-7` (Most capable, highest cost)
- `claude-sonnet-4-6` (Balanced, default)
- `claude-haiku-4-5` (Fastest, most affordable)

## JIDRA Context Format

The validator accepts JIDRA context as JSON containing:

```json
{
  "method": "com.example.PaymentService#processPayment",
  "source": "... method source code ...",
  "class_context": "... class description ...",
  "calls": [
    {"target": "PaymentGateway#charge", "line": 13},
    {"target": "NotificationService#sendConfirmation", "line": 21}
  ],
  "called_by": [
    "OrderService#checkout",
    "RefundService#processRefund"
  ]
}
```

## Features

### 1. Load Real Graph Data
- Specify `--graph /path/to/graph.jsonl` to load actual code analysis
- Browse available methods from the graph
- Search for specific methods by name

### 2. Dynamic Context Building
- Automatically extracts method source from graph records
- Builds JIDRA context with:
  - Method source code (truncated to 2000 chars)
  - Call graph (methods it calls)
  - Method name and file path
- Falls back to demo context if no graph loaded

### 3. Interactive Method Selection
