# validate_jidra_analysis.py — Summary

## What It Does

An **interactive validator** that compares Claude API token usage and costs when using JIDRA-enhanced context vs raw queries.

For each user query, it makes **two parallel API calls**:
1. **WITH JIDRA context** — Focused method information extracted from graph
2. **WITHOUT JIDRA context** — Raw query only

Then displays a comparison table showing token reduction and cost savings.

---

## Key Updates from Original Script

✅ **User-Provided Inputs**
- `--graph /path/to/graph.jsonl` — Load real code analysis data
- `--repo /path/to/codebase` — Specify repository path
- `--model` — Choose model (opus-4-7, sonnet-4-6, haiku-4-5)

✅ **Dynamic Context Building**
- Loads methods directly from graph.jsonl JSONL records
- Extracts method source, calls, class context automatically
- Falls back to demo context if no graph loaded

✅ **Interactive Method Selection**
- Browse available methods from the graph
- Search for specific methods by name
- Auto-builds JIDRA context from selected method

✅ **Flexible Menu**
- Different options depending on whether graph is loaded
- Can set/change graph path at runtime
- Switch models mid-session

## Usage

### Command-Line Options

```bash
python3 validate_jidra_analysis.py [options]
```

| Option | Description |
|--------|-------------|
| `--graph PATH` | Path to graph.jsonl file |
| `--repo PATH` | Path to repository/codebase |
| `--model MODEL` | Choose model: claude-opus-4-7, claude-sonnet-4-6, claude-haiku-4-5 |
| `-h, --help` | Show help and exit |

### Examples

**Load the local validated graph:**
```bash
python3 validate_jidra_analysis.py --graph jidra/output/graph_validated.jsonl
```

**With custom repo and model:**
```bash
python3 validate_jidra_analysis.py \
  --graph /path/to/graph.jsonl \
  --repo /path/to/repo \
  --model claude-sonnet-4-6
```

**Demo mode (no graph needed):**
```bash
python3 validate_jidra_analysis.py
```

## Testing Tips

### Compare Models
Run the same query with different models to see cost/performance tradeoffs:
```
Query: "Analyze this method for bugs"
- Opus-4-7: 1,523 tokens, $0.024
- Sonnet-4-6: 1,456 tokens, $0.005
- Haiku-4-5: 1,389 tokens, $0.001
```

### Test Different Methods
Pick diverse methods from the graph to see how complexity affects token usage:
- Simple getters/setters
- Complex business logic
- Integration methods with many calls

### Measure Cumulative Savings
Run 5-10 queries, then view the summary to see total token and cost savings.

## Notes

- All API calls are **real calls to Claude API** (actual token usage measured)
- Pricing updated as of June 2026
- Thinking tokens cost 3x their output token rate
- Graph files exclude source code beyond 2000 chars to avoid context bloat
- Script is interactive and waits for user input
